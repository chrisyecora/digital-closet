# Digital Closet - AI-Powered Closet Management

This project is a comprehensive closet management system utilizing machine learning for image processing and vector search. It features a FastAPI backend, a dedicated machine learning worker, and a React Native mobile application.

## 🌟 The Vision

Digital Closet is a mobile-first iOS application that uses AI-powered computer vision to build an intelligent, living wardrobe tracker. It replaces manual logging with a daily outfit photo, using AI to identify, categorize, and match garments to previous wear events.

### Core Pillars
- **Daily Photo Logging**: Snap a single photo of your outfit to update your closet history.
- **AI Clothing Detection (YOLO)**: Automatically detect individual items (top, bottom, shoes, etc.).
- **Instance Matching (CLIP + pgvector)**: Identifies if a detected item is the *exact same physical garment* seen before.
- **Intelligent Dashboard**: Surfaces wear frequency, dormancy alerts ("closet bloat"), and co-wear patterns.

## Project Overview

*   **API (`/api`)**: A FastAPI-based service handling business logic, user management, and health checks.
*   **Worker (`/worker`)**: A high-performance Python service dedicated to ML tasks, including object detection (YOLO) and image embedding (CLIP).
*   **Mobile (`/mobile`)**: A React Native application built with **Expo** for user interaction, optimized for iOS.
*   **Database**: PostgreSQL with the `pgvector` extension for efficient vector similarity searches.
*   **Message Queue**: SQS (ElasticMQ) for asynchronous task processing between the API and Worker.
*   **Infrastructure**: Managed locally via Docker Compose, including PostgreSQL and SQS.

## Tech Stack

*   **Backend**: Python 3.12, FastAPI, SQLAlchemy, Alembic.
*   **Worker/ML**: PyTorch (CPU), Ultralytics YOLO, OpenAI CLIP.
*   **Mobile**: React Native (**Expo**/iOS).
*   **Infra**: Docker, Docker Compose, ElasticMQ (SQS), PostgreSQL (pgvector).
*   **Dev Tools**: Poetry (Python dependency management).

## Local Development

### Infrastructure (Docker)
Start the local database and SQS emulator:
```bash
docker compose up -d
```

### API Service
```bash
cd api
poetry install
poetry run uvicorn main:app --reload
```

### Worker Service
The worker requires a specific setup on Intel Macs due to PyTorch/Poetry incompatibilities. ML libraries are installed via `pip` into the Poetry environment.
```bash
cd worker
poetry env use python3.12
poetry install
# Note: ML dependencies (torch, torchvision, ultralytics, clip) are side-loaded via pip
poetry run python main.py
```

### Mobile App (iOS)
```bash
cd mobile/DigitalCloset
npx expo start
```

## Foundational Mandates

1.  **Design First**: Always consult the relevant LLD in `docs/llds/` or spec in `docs/specs/` before making code changes. If a change is significant and undocumented, create or update the corresponding LLD first.
2.  **Type Safety**: Use Pydantic models for all API requests/responses and maintain strict type hinting in Python.
3.  **Environment Isolation**: Use `.env` files for configuration. Never hardcode secrets or environment-specific URLs.
4.  **Database Migrations**: All schema changes must be performed via Alembic migrations (stored in `migrations/`).
5.  **Vector Consistency**: All item embeddings generated by the worker must be compatible with the `pgvector` storage format in the database.
