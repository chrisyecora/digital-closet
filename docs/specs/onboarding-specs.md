# Specifications: Onboarding Flow

## Overview
These specs define the initial user experience for first-time app users, including the value proposition and initial setup.

## Requirements

| ID | Requirement (EARS) | Status |
|----|-------------------|--------|
| **ONBOARD-UI-001** | **When** a user first opens the application, the Mobile Client **shall** display the Welcome screen with the app's value proposition. | [ ] |
| **ONBOARD-UI-002** | **When** a user starts the signup process, the Mobile Client **shall** request Camera permissions and explain their use for wardrobe tracking. | [ ] |
| **ONBOARD-UI-003** | **When** a user starts the signup process, the Mobile Client **shall** request Notification permissions to alert them when outfits are processed. | [ ] |
| **ONBOARD-UI-004** | **When** the permissions are handled, the Mobile Client **shall** offer an optional "Seed Your Closet" step to manually add items. | [ ] |
| **ONBOARD-UI-005** | **Where** a user skips the seeding step, the Mobile Client **shall** direct them to the Home screen's empty state. | [ ] |
| **ONBOARD-UI-006** | **Where** the user has not taken any outfit photos, the Home screen **shall** display an empty state illustration with a call-to-action to take the first photo. | [ ] |
